Work in Progress. For now, the user is encouraged to try to match colors in the rainbow by selecting a color by clicking on the visible light spectrum and finding the right slider combination to match it. The left and middle panels show the mix of slider colors (they are the same, the left just shows better how the colors mix together), and the goal is to match them to the color on the right panel. A correct match will cause the color to appear in the cloud and for a random new color to be selected for matching (though the user may select a new color manually if they wish). The user may also interact with the cloud itself to see other angles.

If the user would like to see the full spectrum when complete, they may select "Next". By clicking "Next" more times, the user can discover future stages of the demonstration. First, is a cloud which can contain most regularly all color spaces defined the way sRGB and Display P3 are, and is scaled using CIE 1931 RGB basis vectors. Later stages include a chromaticity diagram derived from the CIE 1931 RGB color space, the standard chromaticity diagram, the previous cloud but using XYZ basis vectors, and combinations of the previous states but with the Display P3 and sRGB spaces explicitly highlighted.

For best experience, please use desktop for the demonstration.

Note for the curious: when a set of colors is too saturated to display accurately (in this demonstration: the visible light spectrum [with the prism/cloud containing it] and the cloud of all "regularly defined" color spaces [with that which is derived from it, like the chromaticity diagram]), an approximation is made by mixing it with an optimal gray to find the most saturated displayable version of that set of colors. In the future it will be clearer how this approximation is derived.
